## 1. Stan systemu / System State S(t)

### Definicja / Definition

```
S(t) = ( h(t), τ(t), H(t) )
```

gdzie / where:

**h(t) ∈ ℝ^d**  
PL: Wektor stanu ukrytego. Przechowuje "co system wie" w kroku t.
    Inicjalizowany jako h(0) = 0 ∈ ℝ^d.  
EN: Hidden state vector. Stores "what the system knows" at step t.
    Initialized as h(0) = 0 ∈ ℝ^d.

**τ(t) ∈ ℕ**  
PL: Licznik kroków relacyjnych. τ(0) = 0. Rośnie o 1 przy każdym evencie.
    Nie jest timestampem — jest indeksem w sekwencji faktów relacyjnych.  
EN: Relational step counter. τ(0) = 0. Increments by 1 per event.
    Not a timestamp — it is an index in the sequence of relational facts.

**H(t) = { E(0), E(1), ..., E(t-1) }**  
PL: Historia eventów — pełna sekwencja faktów relacyjnych do kroku t.
    H(0) = {} (pusta). Rośnie przy każdym kroku.  
EN: Event history — complete sequence of relational facts up to step t.
    H(0) = {} (empty). Grows at each step.

### Uwaga / Note

PL: S(t) jest stanem WZGLĘDEM wewnętrznego obserwatora systemu.
    Nie jest absolutnym opisem świata — jest relacyjnym.
    To jest bezpośrednie przełożenie RQM Rovellego:
    stan istnieje względem systemu który go rejestruje.

EN: S(t) is the state RELATIVE TO the system's internal observer.
    It is not an absolute description of the world — it is relational.
    This is a direct translation of Rovelli's RQM:
    a state exists relative to the system that registers it.

## 2. Event relacyjny / Relational Event E(t)

### Definicja / Definition

```
E(t) = ( m(t), p(t), φ(t) )
```

gdzie / where:

**m(t) ∈ {0, 1}**  
PL: Wynik pomiaru kwantowego rule_qubit w kroku t.
    Generowany przez Qiskit — stochastyczny podczas generacji,
    deterministyczny podczas rekonstrukcji (znamy historię).
    m(t) = 1 → most CZ odpalił → sieć fizycznie zareagowała.
    m(t) = 0 → most nie odpalił → sieć pozostała niezmieniona.
    To jest "fakt relacyjny" w sensie Rovellego.  
EN: Quantum measurement outcome of rule_qubit at step t.
    Generated by Qiskit — stochastic during generation,
    deterministic during reconstruction (history is known).
    m(t) = 1 → CZ bridge fired → network physically responded.
    m(t) = 0 → bridge did not fire → network unchanged.
    This is the "relational fact" in Rovelli's sense.

**p(t) ∈ [0, 1]**  
PL: P(m=1) przed pomiarem — prawdopodobieństwo Born'a z amplitud stanu
    kwantowego. Mierzy "ile informacji kwantowej" niosło zdarzenie.
    p(t) bliskie 0.5 → maksymalna niepewność → maksymalna informacja.
    p(t) bliskie 0 lub 1 → stan był prawie zdeterminowany.  
EN: P(m=1) before measurement — Born probability from quantum state
    amplitudes. Measures "how much quantum information" the event carried.
    p(t) near 0.5 → maximum uncertainty → maximum information.
    p(t) near 0 or 1 → state was nearly determined.

**φ(t) ∈ ℝ^n**  
PL: Fazy Page-Woottersa dla kroku t — wektor faz z Hamiltoniana gwiazdowego.
    φ(t) = relational_phases(t) z PageWoottersEngine.
    Zapewnia że krok t jest fizycznie odróżnialny od kroku t+1
    nawet gdy m(t) = m(t+1).  
EN: Page-Wootters phases for step t — phase vector from star Hamiltonian.
    φ(t) = relational_phases(t) from PageWoottersEngine.
    Ensures that step t is physically distinguishable from step t+1
    even when m(t) = m(t+1).

### Kluczowa własność / Key property

PL: E(t) jest "faktem relacyjnym" jeśli spełnia:
    1. Powstaje z interakcji (pomiaru) — nie z zewnętrznego sygnału czasowego
    2. Ma konsekwencję fizyczną (most) — nie jest tylko odczytem
    3. Jest nieodwracalny — po zapisaniu do H(t) nie zmienia się

EN: E(t) is a "relational fact" if it satisfies:
    1. It arises from interaction (measurement) — not external time signal
    2. It has physical consequence (bridge) — it is not merely a readout
    3. It is irreversible — once recorded in H(t) it does not change

## 3. Funkcja aktualizacji / Update Function F

### Definicja / Definition

```
S(t+1) = F( S(t), E(t) )
```

rozwinięcie / expanded:

```
h(t+1)   = f_h( h(t), e(t) )        [aktualizacja stanu ukrytego]
τ(t+1)   = τ(t) + 1                 [inkrement licznika]
H(t+1)   = H(t) ∪ { E(t) }          [rozszerzenie historii]
```

gdzie / where:

```
e(t) = embed( m(t), p(t), φ(t) )    [embedding eventu do ℝ^d]
```

### Funkcja f_h — warianty / variants

**Wariant A — GRU-like (v1, minimalna)**

```python
e(t)    = Linear(concat(m(t), p(t), φ(t)))   # embed event → ℝ^d
r(t)    = σ( Wr · concat(h(t), e(t)) )       # reset gate
z(t)    = σ( Wz · concat(h(t), e(t)) )       # update gate
h̃(t)   = tanh( W  · concat(r(t)*h(t), e(t)))# candidate
h(t+1) = (1-z(t))*h(t) + z(t)*h̃(t)          # new state
```

PL: GRU jest wybrane bo:
    - bramka reset (r) kontroluje ile poprzedniego stanu zostaje
    - bramka update (z) kontroluje jak mocno event zmienia stan
    - bez bramek: każdy event nadpisuje stan → brak pamięci trajektorii
    Bramki są analogiem "selektywnej uwagi na historię".

EN: GRU is chosen because:
    - reset gate (r) controls how much previous state survives
    - update gate (z) controls how strongly event changes state
    - without gates: each event overwrites state → no trajectory memory
    Gates are the analog of "selective attention to history".

**Wariant B — Attention over history (v2, przyszły)**

```
h(t+1) = Attention( h(t), H(t) )
```

PL: Zamiast rekurencji — uwaga nad całą historią eventów.
    Bardziej wyrazisty relacyjnie, droższy obliczeniowo.
    Zostawione na v2.

EN: Instead of recurrence — attention over full event history.
    More explicitly relational, more expensive.
    Reserved for v2.

## 4. Pomiar temporal awareness / Measuring temporal awareness

PL: Jak sprawdzamy czy RTANA "działa"?
    Potrzebujemy mierzalnej różnicy behawioralnej.

EN: How do we check if RTANA "works"?
    We need a measurable behavioral difference.

### Metryka 1 — Step prediction accuracy

```
task: given S(t), predict τ(t) mod N
```

PL: Czy system "wie" na którym kroku jest?
    Model z relacyjnym zegarem powinien lepiej przewidywać
    swój własny krok niż model bez stanu.
    Baseline: losowe zgadywanie → 1/N accuracy.
    RTANA target: > 1/N dla N = {4, 8, 16}.

EN: Does the system "know" which step it is on?
    A model with relational clock should better predict
    its own step than a stateless model.
    Baseline: random guessing → 1/N accuracy.
    RTANA target: > 1/N for N = {4, 8, 16}.

### Metryka 2 — History sensitivity

```
task: given S(t) from two trajectories T1, T2
      with identical recent events but different H(t),
      predict which trajectory produced S(t)
```

PL: Czy stan S(t) zawiera informację o historii?
    Jeśli dwie trajektorie mają takie same ostatnie eventy
    ale różną historię, stan RTANA powinien je rozróżniać.
    Model bez historii nie może — jego stan zależy tylko
    od ostatniego eventu.

EN: Does S(t) carry information about history?
    If two trajectories have identical recent events
    but different history, RTANA state should distinguish them.
    A stateless model cannot — its state depends only
    on the last event.

### Metryka 3 — Entropy of relational timeline

```
H_shannon = -Σ p(m=k) log p(m=k)   k ∈ {0,1}
```

PL: Entropia osi czasu — już mierzona w RQTE.
    Target: H ≈ 1.0 bit (maksymalna losowość pomiarów).
    BUTTERFLY class pokazał H = 0.811 — system z historią
    jest mniej losowy. To jest obserwacja do zbadania.

EN: Timeline entropy — already measured in RQTE.
    Target: H ≈ 1.0 bit (maximum measurement randomness).
    BUTTERFLY class showed H = 0.811 — system with history
    is less random. This is an observation to investigate.

## 5. Minimalna implementacja / Minimal implementation

### Stack

```
Qiskit     → generuje E(t): pomiary, fazy PW, bridge
PyTorch    → implementuje F: GRU update, embedding
Python     → orkiestracja, metryki, logging
```

### Pseudokod / Pseudocode

```python
# Inicjalizacja / Initialization
pw     = PageWoottersEngine(N_clock=4, N_sys=5, J=1.0)
gru    = RelationalGRU(d_hidden=64, d_event=pw.N_sys + 2)
S      = State(h=zeros(64), tau=0, history=[])
sv     = GraphState(n=5, edges=STAR_EDGES)

# Pętla relacyjna / Relational loop
for t in range(N_STEPS):

    # Qiskit: generuj event
    sv, event = relational_step(sv, pw, t, rng)
    # event = (m, p, phi)

    # PyTorch: aktualizuj stan
    e = gru.embed(event)
    h_new = gru.update(S.h, e)

    # Aktualizuj S(t) → S(t+1)
    S = State(
        h       = h_new,
        tau     = S.tau + 1,
        history = S.history + [event]
    )

# Ewaluacja / Evaluation
tau_pred = predict_step(S.h)          # metryka 1
traj_id  = predict_trajectory(S.h)    # metryka 2
entropy  = timeline_entropy(S.history) # metryka 3
```

## 6. Co to NIE jest / What this is NOT

PL:
- Nie jest modelem językowym
- Nie jest agentem
- Nie jest symulacją świadomości
- Nie jest "lepszą pamięcią" do LLM (to jest osobny projekt)

To jest minimalna architektura do testowania jednej hipotezy:
czy stan kwantowo-inspirowany z relacyjną historią eventów
jest mierzalnie inny od stanu bez tej historii.

EN:
- Not a language model
- Not an agent
- Not a simulation of consciousness
- Not "better memory" for LLMs (that is a separate project)

This is a minimal architecture to test one hypothesis:
whether a quantum-inspired state with relational event history
is measurably different from a state without that history.
